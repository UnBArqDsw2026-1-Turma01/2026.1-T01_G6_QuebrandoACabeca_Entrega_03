# 3.5. Iniciativas Extras - Padrões de Projeto

Breve relato sobre as Iniciativas Extras realizadas pela equipe, no escopo da entrega.

## 🎨 Desenvolvimento do Frontend

Como iniciativa extra, a equipe desenvolveu o frontend completo da aplicação em **React + TypeScript**, contemplando todas as telas do fluxo do usuário da autenticação até a tela de vitória, sem depender de comunicação com o backend até o momento.

O trabalho incluiu:

- Criação do Fluxograma do comportamento esperado do Software
- Criação das telas documentadas no Figma de acordo com o fluxograma previamente produzido
- Criação de todas as telas com seus respectivos arquivos `.tsx` e `.css`
- Implementação da navegação entre telas via **React Router DOM**
- Comportamentos interativos funcionais (drag & drop das peças, toggles, sliders, modais)
- Estilização consistente com variáveis CSS globais e design system próprio

> 🔗 Comprobatório: [Documentação do Processo de Desenvolvimento do Frontend](/Documentacao/FrontEnd.md)

## 🌐 Desenvolvimento do Gerenciador de Partidas Multijogador

Mais uma contribuição adicional sugerida e iniciada foi a funcionalidade para
partidas multijogador no *backend*, primariamente centrada em uma classe *singleton* `GameSessionManager`. Ela é responsável pelo gerenciamento da conexão entre os jogadores através do servidor, visando-se oferecer (pelo menos) métodos para entrar em uma sala, enviar a jogada do usuário e um jogador ouvir as jogadas do oponente.

Como já sugerido pelo uso do padrão *singleton*, a ideia é que exista uma única instância dela em toda a aplicação.

Para viabilizar a materialização deste gerenciador mesmo no repositório *toy*, foi necessário preparar algumas classes de dependência no *backend*, como:

- As sessões nominais que o gerenciador irá gerenciar (`GameSession`);
- O tabuleiro virtual (`PuzzleBoard`) e as jogadas (`Move`) a serem referenciadas pela `GameSession`;
- E as peças (`Piece`) a serem movidas numa jogada.
