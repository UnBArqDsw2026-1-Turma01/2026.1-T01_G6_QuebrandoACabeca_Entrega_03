## 3.2.3.Justificativa

O GOF estrutural composite foi importante para decidirmos como seria implementado a lógica de jogar o quebra-cabeça. Tivemos varias discussões em grupo de como fariamos a junção das peças, se seria feita com o local delas preestabelecido ou tentano juntar elas liveremente. Porém essa última opção estava mais complicada por não termos conhecimento de como implementá-la. Então, com a descoberta do composite, possibilita a implementação da lógica de vários conjuntos de peças separados para depois construir o quebra-cabeça inteiro

## 3.1.5.Análise GRASP

| Princípio GRASP | Como se manifesta no diagrama |
|---|---|
| **Polimorfismo** | `mover()`, `verificarColisao()` e `getPosicao()` têm implementação polimórfica — o cliente nunca distingue folha de composto |
| **Especialista na Informação** | `PecaUnica` é especialista de sua posição e formato (`formatoPeca: int[4]`); `Composite` é especialista da lista de filhos (`componentes: List<Component>`) |
| **Baixo Acoplamento** | O cliente referencia apenas `Component` — zero acoplamento com `PecaUnica` ou `Composite` |
| **Variações Protegidas** | Novos tipos de agrupamento entram sem alterar cliente nem a interface |
| **Alta Coesão** | `PecaUnica` cuida de si mesma; `Composite` cuida exclusivamente de gerenciar e delegar |


## 3.1.6.Análise de complexidade

As operações `mover(deltaX, deltaY)` e `verificarColisao()` no `Composite` percorrem recursivamente todos os filhos, resultando em complexidade O(n), onde n é o total de componentes na subárvore. No caso de `verificarColisao()`, se a verificação ocorrer entre todos os pares de filhos, a complexidade pode atingir O(n²) — sendo recomendável restringir a verificação à bounding box do grupo para mantê-la em O(n). Operações sobre `PecaUnica` são O(1).


## 3.1.7.Implementação


O padrão Composite foi implementado no módulo `composite/` do backend, composto pelos seguintes arquivos:

- `component.py` — interface Component com os métodos `mover()`, `verificarColisao()` e `getPosicao()`
- `peca_unica.py` — folha PecaUnica, representa cada peça individual com posição atual, posição correta e tolerância de encaixe
- `composite.py` — Composite, agrupa peças já encaixadas tratando-as como uma única unidade, suportando grupos de grupos
